Roles utilisateurs Firebase
Dans Firebase → Realtime Database → Regles, colle ces regles pour proteger le backend :
{
  "rules": {
    "users": {
      ".read": "auth != null",
      ".write": "auth != null"
    },
    "employees": {
      ".read": "auth != null",
      ".write": "auth != null && root.child('users').child(auth.uid).child('role').val() === 'admin'"
    },
    "planning": {
      "shifts2026": { ".read": "auth != null", ".write": "auth != null" },
      "shifts2025": { ".read": "auth != null", ".write": "auth != null" },
      "absences": {
        ".read": "auth != null && root.child('users').child(auth.uid).child('role').val() === 'admin'",
        ".write": "auth != null && root.child('users').child(auth.uid).child('role').val() === 'admin'"
      },
      "lastUpdate": { ".read": "auth != null", ".write": "auth != null" }
    }
  }
}
Ces regles garantissent que les sous-chefs ne peuvent pas lire ni ecrire les absences meme via une requete directe.
